## Main Idea

将三位哲学家的认知方法整合为一套"思维清洁系统"：维特根斯坦负责发现哪里模糊、苏格拉底负责追问出能说清楚的答案、波兰尼负责处理怎么追问都说不出来的默会知识。这套方法可用于写 prompt、做决策、带团队、对需求。

## Key Details

### 第一层：维特根斯坦 — 命题分解

**核心观点**：语言是思想的边界，说不清楚一件事本质是没想清楚。

**方法**：开口前把要说的话拆成三类——
- **F（事实）**：可验证的客观信息
- **D（欲望）**：希望达成的状态
- **Q（困惑）**：自己也不确定的部分

例："帮我优化一下这篇文章"
- F = 有一篇已经写好的文章（清楚）
- D = "想让它变好"，但"好"是什么意思？（模糊）
- Q = 不确定到底该改什么（更模糊）

### 第二层：苏格拉底 — 反诘法

**核心观点**：大多数时候你不是不知道答案，你是没有被正确地问过。

**方法**：不给答案，只问。问直到你自己把答案说出来。

**典型追问链**：
1. "优化"是改措辞、改结构、还是改论点？
2. 给谁看的？
3. 什么样的输出你会觉得"对了"？
4. "像麦肯锡报告那种感觉" → 追问到极限了，说不出来了

### 第三层：波兰尼 — 默会知识

**核心观点**：有些知识怎么问都说不出来，因为它根本不是语言能承载的。

**From-To Structure**：认知是从大量细碎辅助线索"到"整体判断的过程——瞬间完成、无意识的。注意力一旦转向细节本身，整体感觉反而会崩塌（类似骑自行车，越想怎么骑越不会骑）。

**三个传递策略**：
1. **示范**：给一个"对"的例子，让对方从里面提取说不出来的偏好
2. **反面**：说不出来要什么，就说不要什么（否定句比肯定句更容易逼出默会知识）
3. **行为提取**：从过去的行为模式中反向提取隐含规则

## Method / Approach

作者整合三位哲学家观点，形成 Clarify 流程工具。核心洞察是：模糊念头 → 维特根斯坦扫描（发现模糊） → 苏格拉底追问（挖掘可问出的） → 波兰尼感应（处理问不出的） → 清晰可执行的想法。

## Results / Evidence

**完整流程**：
```
模糊念头
  ↓
维特根斯坦（扫描）→ 拆解 → 发现哪些部分模糊
  ↓
苏格拉底（挖掘）→ 追问 → 把能问出来的都问出来（但总有些"感觉"问不出来）
  ↓
波兰尼（感应）→ 示范 / 反面 / 行为提取 → 处理说不出来的部分
  ↓
清晰可执行的想法
```

**应用场景**：写 prompt、做决策、带团队、对需求、自我对话

## Limitations

- 这是作者个人经验总结，没有严格的对照实验验证
- 三层方法论需要练习才能熟练掌握
- 对于高度技术性的问题，默会知识可能需要更多具体示例
