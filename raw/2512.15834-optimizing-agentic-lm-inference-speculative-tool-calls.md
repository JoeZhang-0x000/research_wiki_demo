# Optimizing Agentic Language Model Inference via Speculative Tool Calls

**Authors**: Daniel Nichols, Prajwal Singhania, Charles Jekel, Abhinav Bhatele, Harshitha Menon

**Submitte**: arXiv, 17 Dec 2025

**Subjects**: Programming Languages (cs.PL); Artificial Intelligence (cs.AI); Distributed, Parallel, and Cluster Computing (cs.DC); Performance (cs.PF); Software Engineering (cs.SE)

## Abstract

Language models (LMs) are becoming increasingly dependent on external tools. LM-based agentic frameworks frequently interact with their environment via such tools to search files, run code, call APIs, etc. Further, modern reasoning-based LMs use tools such as web search and Python code execution to enhance their reasoning capabilities. While tools greatly improve the capabilities of LMs, they also introduce performance bottlenecks during the inference process.

In this paper, we introduce novel systems optimizations to address such performance bottlenecks by speculating tool calls and forcing sequences to remain resident in the inference engine to minimize overheads. Our optimizations lead to throughput improvements of several hundred tokens per second when hosting inference for LM agents.

We provide a theoretical analysis of our algorithms to provide insights into speculation configurations that will yield the best performance. Further, we recommend a new "tool cache" API endpoint to enable LM providers to easily adopt these optimizations.

## Key Contributions

1. **Speculative Tool Calls**: Predicting and pre-executing tool calls before the LLM explicitly requests them
2. **Sequence Residency**: Keeping tool sequences resident in the inference engine to minimize overhead
3. **Theoretical Analysis**: Analysis of speculation configurations for optimal performance
4. **Tool Cache API**: A proposed new API endpoint for LM providers to adopt these optimizations

## Background

Language models increasingly rely on external tools (file search, code execution, API calls, web search, etc.) for agentic tasks. While tools greatly expand LLM capabilities, they introduce performance bottlenecks during inference because:

1. Each tool call requires context switching between LLM and tool execution
2. Tool execution latency adds directly to end-to-end request time
3. The LLM must wait for each tool to complete before generating the next token

Modern reasoning-based LMs (e.g., those using ReAct, Toolformer, etc.) use tools like web search and Python execution to enhance reasoning, but this exacerbates the latency problem.

## Proposed Approach

The paper proposes two main optimizations:

### 1. Speculative Tool Calls

The core idea is to predict which tools the LLM will need before it explicitly requests them, and pre-execute them in parallel with LLM generation. This is analogous to CPU branch prediction and speculative execution.

Key challenges:
- Predicting which tool will be needed
- Predicting the correct parameters for the tool
- Ensuring correctness when speculation is wrong

### 2. Sequence Residency

Keeping frequently-used tool sequences resident in memory rather than reloading them for each request. This reduces the overhead of:
- Loading tool definitions
- Initializing tool environments
- Establishing connections to external services

## Theoretical Analysis

The paper provides theoretical analysis of:
- When speculation is beneficial (based on tool execution time vs prediction accuracy)
- Optimal speculation depth (how many tools to speculatively execute)
- Trade-offs between speculation accuracy and performance gains

## Tool Cache API

The authors propose a new "tool cache" API endpoint that allows:
- Pre-loading of tool definitions and environments
- Persistent connections to external services
- Caching of intermediate results between related tool calls

This would enable LM providers to easily adopt the optimizations without changing their core architecture.

## Results

The optimizations lead to throughput improvements of **several hundred tokens per second** when hosting inference for LM agents. The exact improvements depend on:
- The prediction accuracy of the speculation system
- The execution time of the tools being speculated
- The complexity of tool parameter derivation
