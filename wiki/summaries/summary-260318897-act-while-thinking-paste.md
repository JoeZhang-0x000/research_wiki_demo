## Main Idea

PASTE (Pattern-Aware Speculative Tool Execution) accelerates LLM agent serving by exploiting two key observations: (1) agent tool invocations follow stable, recurring control-flow patterns despite semantic diversity, and (2) tool parameters are often derivable from outputs of prior tools rather than hallucinated from scratch. By formalizing these patterns as **Pattern Tuples (C, T, f, p)** and executing speculatively on slack resources, PASTE reduces average task completion time by **48.5%** and improves tool throughput by **1.8x**.

## Key Details

**Problem characterization (3 benchmarks: Deep Research Bench, SWE-Bench, ScholarQA):**
- Tool execution = 35%-61% of total request time
- Coding tasks: 60% tool latency
- Deep research tasks: 50% tool latency
- Scientific tasks: 36% tool latency

**Pattern Tuple (C, T, f, p):**
- **C** (Context): event signatures—sequence of tool types that identify stable control flows
- **T** (Tool): predicted next tool to invoke
- **f** (Function): symbolic value mapping that derives tool parameters from prior outputs
- **p** (Probability): confidence score

**Control-flow patterns observed:**
- Edit-Verify (coding): 55% of file_editor calls immediately followed by pytest/python execution
- Locate-Examine (coding): 38% of grep calls immediately followed by file_editor
- Search-Visit (research): 51% of search calls immediately followed by web_fetch for top 3-4 URLs
- Pre-fetch (research): 20% of web_fetch calls trigger pre-fetching of embedded URLs

**Parameter derivation:**
- In 95% of web tasks: URL for visit tool is a strict substring of preceding search tool's JSON output
- Agent doesn't "hallucinate" parameters—it derives them from prior tool outputs

## Method / Approach

**Pattern Mining (2 phases):**
1. **Structural Mining**: identifies recurring tool-call sequences from historical agent traces using n-gram analysis
2. **Symbolic Dependency Inference**: infers value mapping functions via symbolic execution (how output of tool N becomes input to tool N+1)

**Online Pattern Prediction:**
1. Context Matching: matches current tool sequence against known patterns
2. Function Evaluation: applies value mapping function to derive parameters
3. Probabilistic Hint Generation: outputs (tool, parameters, confidence)

**Opportunistic Speculative Scheduling:**
- **Authoritative jobs**: generated by agent, correctness-critical, strict priority + preemption
- **Speculative jobs**: generated by PASTE, best-effort, run only on slack resources (idle compute/memory/IO)
- **Dynamic slack budget**: throttles speculation so it never competes with authoritative work
- **Instantaneous preemption**: mispredictions never slow down authoritative progress
- **Promotion protocol**: when authoritative request matches running speculative job, it's immediately promoted and committed

## Results / Evidence

| Metric | Improvement |
|--------|------------|
| Average task completion time | **-48.5%** |
| Tool execution throughput | **1.8x** |

Evaluated on: Deep Research Bench, SWE-Bench, ScholarQA with Gemini-2.5, GPT-5.2, Qwen-DeepResearch-30B on 8x NVIDIA A100-80G GPUs.

## Limitations

- Requires historical traces to mine patterns before deployment
- Pattern accuracy depends on workload stability—if agent behavior shifts significantly, patterns may become stale
- The paper does not address how to handle novel tool combinations not seen in training data
- Speculation overhead (pattern matching, function evaluation) adds small constant cost that must be amortized
