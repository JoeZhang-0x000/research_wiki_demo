## Main Idea

Tw93 created **Waza** (Japanese: わざ, "technique/move"), an open-source skill set of 8 markdown-only commands that distill the essential capabilities a good engineer needs in the AI era: Thinking, Design, Troubleshooting, Checking, Reading, Writing, Learning, and Maintaining. The key insight: rather than accumulating endless tools, engineer taste and a fixed set of composable skills outperform constant tool churn.

## Key Details

**Waza's 8 skills:**

1. **/think** — Thinking: question the problem before starting, stress-test the plan, clarify architecture before letting AI execute. Guards against going fast in the wrong direction.

2. **/design** — Design: ensure AI output has taste and clear design direction, not just functional correctness. Guards against generic, aesthetically undifferentiated output.

3. **/hunt** — Troubleshooting: approach bugs systematically, find root cause first, fix properly in one pass. Guards against the "change-something-and-try-again" loop.

4. **/check** — Checking: review diffs before merging, auto-fix what can be fixed, group judgment-required items, validate with evidence not gut feeling.

5. **/read** — Reading: read primary sources directly (URL or PDF → clean Markdown), bring into workflow. Guards against secondhand summary dependency.

6. **/write** — Writing: communicate clearly to the right audience. Strong technical skills are wasted without clear explanation.

7. **/learn** — Learning: collect → digest → outline → draft → polish → publish. Output-driven learning for unfamiliar fields. Guards against passive reading without retention.

8. **/health** — Maintaining: toolchain checkups. When CLAUDE.md, rules, hooks, or MCP configs start feeling off, run /health to quickly identify the problem.

**Core philosophy:**
- "Installing this set is enough. I don't need to keep messing with more skills."
- "Just the right amount, not more, not less."
- "Product idea with engineer skill is better" (from reply)
- GitHub: github.com/tw93/waza — 100% markdown, no code

## Method / Approach

Waza is a curated, opinionated skill set rather than a comprehensive toolkit. Each skill encapsulates a disciplined engineering habit:
- Fixed interface (slash command) → consistent invocation
- Markdown-based → no code dependencies, portable across AI coding tools
- Opinionated defaults → reduces decision fatigue

## Limitations

- Skill names (/think, /design, etc.) are generic and may collide with other skill sets (raised in replies, Tw93 acknowledged)
- No code implementation—all markdown, so落地 depends on AI coding tool supporting custom slash commands
- No automated testing or verification of skill quality
